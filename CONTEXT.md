# MediaCookies

MediaCookies 帮助用户判断哪些媒体站点登录数据适合导出，并生成可交给 Media Dock 使用的本地 Cookie 包。

## Language

**Media Dock 用户**:
希望顺利下载受登录状态限制的媒体内容，但不需要理解或管理 Cookie 技术细节的人。
_Avoid_: 小白用户、普通用户

**Cookie 来源**:
一组属于同一媒体服务、可作为整体判断和导出的 Cookie。
_Avoid_: 站点、服务、域名包

**当前来源**:
Media Dock 用户此刻正在访问、并希望用于后续下载的 Cookie 来源。
_Avoid_: 当前标签页、当前域名

**来源授权**:
用户允许 MediaCookies 访问一个 Cookie 来源所需相关域名的明确授权。
_Avoid_: 网站权限、读取权限

**全来源扫描**:
用户在导出工作台中主动按域名查看浏览器内所有可访问 Cookie 的操作，结果可能包含与媒体下载无关的站点。
_Avoid_: 全部 Cookie 模式、全站权限

**媒体站点扫描**:
只显示内置来源及 Media Dock 下载工具（yt-dlp）支持站点的推荐扫描范围。
_Avoid_: 可下载来源、有效来源

**扫描范围**:
决定工作台显示媒体站点还是所有 Cookie 域名的筛选条件，不表示登录状态是否有效。
_Avoid_: 导出状态、登录状态

**临时扫描**:
Cookie 值只在一次检查或导出期间存在于内存，并在操作结束或界面关闭后丢弃的扫描过程。
_Avoid_: 预览缓存、扫描记录

**快速导出**:
识别当前来源、判断导出条件并使用安全默认值生成 Cookie 包的精简流程。
_Avoid_: 简单模式、普通模式

**导出状态**:
快速导出对当前来源是否适合生成 Cookie 包的结论。
_Avoid_: 检测结果、登录态等级

**媒体通行证**:
把当前来源及其导出状态组织成 Media Dock 用户可以立即理解的一份本地凭证摘要。
_Avoid_: 状态卡片、Cookie 卡片

**可以导出**:
当前来源具备预期的登录状态，可以直接生成 Cookie 包。
_Avoid_: 正常、完整

**请先登录**:
当前来源缺少必要的登录状态，应先回到媒体服务完成登录。
_Avoid_: 缺失、失败

**可能失效**:
当前来源的登录状态不完整或即将过期，可以继续导出，但结果存在失败风险。
_Avoid_: 部分、警告

**需确认**:
导出工作台无法验证某个来源的登录标记，或发现 Cookie 临近过期时使用的汇总状态。
_Avoid_: 可能失效、扫描失败

**当前网页不支持**:
当前访问内容无法对应到可快速导出的 Cookie 来源。
_Avoid_: 未匹配、未知来源

**导出工作台**:
面向扫描、检查、筛选和自定义导出的完整操作空间。
_Avoid_: 高级页面、设置页

**常用来源**:
Media Dock 用户保存、并希望在后续导出中快速复用的一组 Cookie 来源偏好。
_Avoid_: 常用配置、来源配置

**Cookie 包**:
一次导出产生的本地文件集合，可由 Media Dock 读取。
_Avoid_: ZIP、导出文件、Cookie 文件

**Cookie 包契约**:
确保现有 Media Dock 可以继续识别 Cookie 包路径、来源 ID 和 Cookie 内容格式的兼容边界。
_Avoid_: ZIP 格式、导入格式

**本地交接**:
MediaCookies 与 Media Dock 通过用户设备上的 Cookie 包交换数据，不建立网络上传或应用直连。
_Avoid_: 同步、直连、发送到 Media Dock
