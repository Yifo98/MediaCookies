# Use TypeScript deep modules and browser adapters

MediaCookies will use a TypeScript and Vite multi-page build without a runtime UI framework. Source recognition, export status, and package generation will live behind small workflow interfaces, while Chrome tabs, Cookies, permissions, downloads, and storage will be concentrated in a browser adapter with an in-memory test adapter. Downloads use a temporary page anchor with an explicit `download` filename instead of `chrome.downloads.download`; this preserves `MediaCookies_YYYY-MM-DD_HH-mm-ss.zip` across Chromium variants and avoids the `downloads` permission. This creates real seams for browser testing and keeps both interfaces free from scattered `chrome.*` calls without adding React, Vue, or utility-CSS framework complexity.
