# 浏览器验收

本文件定义 MediaCookies 重构后的 Chrome 与 Microsoft Edge 验收范围。目标是真实加载、真实权限、真实 Cookie API 和真实下载流程，同时避免把个人登录凭证带入自动化。

## 测试资料

- 自动化和重复验收使用独立浏览器资料目录，不连接日常浏览器资料。
- 默认只注入专用合成 Cookie，不使用真实账号 Cookie。
- 合成数据覆盖可以导出、请先登录、可能失效、当前网页不支持和权限被拒绝。
- 只有合成环境无法复现特定问题时，才允许使用已有登录状态进行人工冒烟测试。
- 任何测试输出、截图和日志都不得包含 Cookie 值。

## 验证层级

1. 通过纯 module 测试验证来源识别、导出状态、过期判断、Netscape Cookie 内容和 Cookie 包契约。
2. 通过内存 browser adapter 验证快速导出与导出工作台的完整工作流。
3. 在独立 Chrome 资料中加载生产 `dist/`，验证当前来源、按需授权、Cookie 读取、本地下载和界面状态。
4. 在 Microsoft Edge 中重复加载、授权、导出与工作台核心流程。
5. 使用 Playwright CLI 或等价的真实浏览器控制方式完成可访问性快照、关键交互、视觉截图和控制台检查。

## 必测场景

- 当前来源已授权且登录状态完整，可以生成向后兼容的 Cookie 包。
- 当前来源未授权，解释用途后申请来源授权。
- 用户拒绝来源授权，界面保持可恢复并说明重新授权方式。
- 当前来源缺少登录状态，显示请先登录且不误报为系统错误。
- 当前来源登录状态不完整或即将过期，显示可能失效但允许继续导出。
- 当前网页不支持，提供返回目标网页或打开导出工作台的路径。
- 导出完成后清除内存中的临时扫描结果。
- 浏览器范围扫描可由快速导出的“识别整个浏览器”卡片或工作台按钮明确发起，并在执行前说明广泛权限；快速入口授权后自动打开工作台，默认执行“媒体站点（推荐）”扫描，“全部 Cookie 域名”只由用户在工作台明确选择。
- 工作台将“媒体站点（推荐）”与“全部 Cookie 域名”明确标为扫描范围，并说明两者使用相同授权；范围不与“可以导出 / 需确认 / 请先登录”状态混为一谈。
- 推荐范围同时覆盖内置来源和仅由 yt-dlp 目录识别的来源（如 Vimeo）；任意 `example.com` Cookie 只在“全部 Cookie 域名”中出现，推荐扫描将其计为“推荐范围外”。
- 预览表格和界面 DOM 不包含 Cookie 值。
- 中文与英文切换由用户主动触发，并记住上次选择。

## 浏览器与视觉检查

- Chrome 是完整验收浏览器；Edge 至少覆盖安装、权限、导出和工作台。
- 其他 Chromium 浏览器只做尽力兼容，不阻塞本次重构。
- 每次重要状态变化后重新获取浏览器快照，避免使用过期元素引用。
- 检查键盘导航、可见焦点、状态提示语义、颜色对比和缩放后的布局。
- 商店截图必须来自真实加载的生产扩展，不使用静态 HTML 或 Chrome API 缺失状态冒充。
- 浏览器截图与轨迹统一保存在项目内 `output/playwright/`。

## 完成条件

- Chrome 核心流程全部通过，控制台没有未解释错误。
- Edge 核心兼容流程通过，或明确记录可复现的平台限制。
- 生成的 Cookie 包由当前 Media Dock 版本成功导入。
- 浏览器实际落盘文件名匹配 `MediaCookies_YYYY-MM-DD_HH-mm-ss.zip`，并与 ZIP 根目录名称一致，不退化为 Blob URL 的 UUID。
- 生产 Manifest 不申请 `downloads` 权限，Chrome Web Store 安装后的扩展仍能完成本地 ZIP 下载。
- 五张商店截图通过尺寸、色彩模式、alpha 和可读性检查。
