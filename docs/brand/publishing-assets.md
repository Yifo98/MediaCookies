# MediaCookies 发布视觉素材

本文件定义 MediaCookies 重构后的图标、Chrome Web Store 素材和 GitHub 展示素材。尺寸与格式以 [Chrome Web Store 官方图片规范](https://developer.chrome.com/docs/webstore/images?hl=zh-cn) 及开发者信息中心的实际上传要求为准。

## 品牌方向

- 公开主名称：MediaCookies。
- 中文副说明：为 Media Dock 导出登录状态。
- 英文副说明：Cookie export for Media Dock.
- `XF` 仅作为开发者或发布者身份使用，不进入主 Logo 和产品标题。
- 视觉概念：本地媒体通行证。
- 选定标志：第一款“通行印记”几何候选。
- 标志由开放圆环、播放负形和青紫通行切片组成。
- 参考现代技术品牌的简洁度与比例纪律，不临摹任何现有品牌标志。
- 禁止使用 Cookie 饼干、锁、钥匙、浏览器窗口、下载箭头和字母缩写作为主图形。

## 品牌家族关系

- Media Dock 3.0 的品牌母体为 “Luminous Workshop / 微光工作室”，桌面应用使用独立的紧凑版 Media Berth 标志。
- MediaCookies 保留自己的“通行印记”，不得直接复用或改画 Media Berth 的主体结构。
- 两者仅共享暖白、石墨、湖蓝、淡紫、克制材质和小尺寸清晰度。
- Media Dock 表达媒体停泊与处理；MediaCookies 表达当前来源的通行状态与本地交接。
- 家族识别来自色彩、材质、比例和排版纪律，不依靠相似轮廓制造关联。

## 图标版本

同一个几何标志提供两种场景版本，不创建两套 Logo：

- 浏览器工具栏版：透明背景，针对 16px 和 32px 优化轮廓、间距和色块分离。
- 商店与展示版：暖白圆角底，正面视角，仅保留轻微阴影。

需要生成以下浏览器图标：

- `icon16.png`：16x16
- `icon32.png`：32x32
- `icon48.png`：48x48
- `icon128.png`：128x128

Chrome Web Store 的 128x128 PNG 使用 96x96 实际方形图标，并在四周各保留 16px 透明内边距。不得额外添加边缘或大阴影，必须同时在浅色和深色背景上保持清晰。

## Chrome Web Store 素材

### 屏幕截图

- 中文与英文共用一套 5 张截图，不分别制作两套本地化截图。
- 中文 3 张：可以导出的快速导出、按需授权或请先登录、带扫描范围说明的导出工作台总览。
- 英文 2 张：导出成功与 Media Dock 本地交接、推荐媒体站点扫描与安全预览。
- 统一使用 1280x800，方角、无填充、全出血。
- 使用 JPEG 或 24 位 PNG；PNG 不得包含 alpha 透明层。
- 截图必须展示真实扩展体验，不使用纯概念图代替产品界面。
- 缩小到 640x400 后，核心状态和主要操作仍须可读。

### 宣传图片

- 必需小宣传图片：440x280。
- 可选横幅：1400x560。
- 宣传图以品牌和核心价值为主，不直接复用产品截图。
- 优先使用无文字构图，避免本地化冲突；画面填满可用区域，缩小一半后仍清楚。

## GitHub 素材

- 社交预览卡：1280x640。
- README 顶部品牌图：独立横版构图，不拿商店截图拉伸代替。
- GitHub 素材可包含项目名称和一句短说明，但必须使用后期排版保证文字准确，不依赖生成模型绘制文字。

## 验收

- 自动检查所有最终文件的像素尺寸、PNG 色彩模式和 alpha 状态。
- 分别预览 16px、32px、48px、128px 和大图展示效果。
- 商店图标必须在浅色、深色背景中检查视觉重量和边缘清晰度。
- 商店截图必须来自真实加载的扩展，并覆盖快速导出和导出工作台的核心流程。
